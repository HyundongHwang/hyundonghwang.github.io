## 🧭 1. Obsidian 소개

### 🔹 1.1 Obsidian이란?

**Obsidian**은 마크다운(Markdown) 문법으로 작성할 수 있는 크로스 플랫폼 노트 애플리케이션입니다. 여타 노트 앱과는 차별화되는 높은 범용성과 생태계 지원으로 출시 직후 많은 호응을 얻으며 메이저 노트 앱 중 하나로 자리매김하는 것에 성공했습니다.

옵시디언은 여러분이 생각하는 방식 그대로 사용 가능한 유연하고 안전한 노트 앱입니다. **생각의 정리와 확장을 도와주는 현존 최고의 노트 앱**이라 볼 수 있습니다.

### ✨ 1.2 주요 특징

#### ▫️ 마크다운 기반
공개된 마크다운 포맷을 사용하여 옵시디언 앱이 아니더라도 아무 편집기로나 문서를 어디서든 읽고 편집할 수 있습니다. 다른 노트 플랫폼으로의 이동이 압도적으로 쉬워집니다.

```markdown
# 제목 1
## 제목 2
**굵은 글씨**
*기울임 글씨*
- 리스트 항목
[링크 텍스트](URL)
```

#### ▫️ 그래프 뷰
옵시디언의 꽃이라고 할 수 있는 **그래프 뷰**가 있으며, 이 기능을 통해 메모와 메모끼리 연결한 그래프를 직관적으로 볼 수 있습니다.

```
[노트 A] ─── [노트 B]
    │            │
    └──── [노트 C]
```

또한 **Obsidian Canvas** 기능이 공개되었으며, 이를 사용해 문서 기반으로 마인드맵, 다이어그램 등을 손쉽게 그릴 수 있습니다.

#### ▫️ 플러그인 생태계
방대한 플러그인 생태계 덕분에 유저들이 개발한 수많은 종류의 플러그인으로 대부분의 단점들을 해소할 수 있습니다.

#### ▫️ 무료 사용
상업적 목적으로 사용하는 것이 아니라면 무료이며, iCloud나 다른 third-party 서비스를 사용하면 무료로 동기화가 가능합니다.

## 🛠️ 2. 설치 및 초기 설정

### 🛠️ 2.1 설치 방법

1. **Obsidian 공식 웹사이트 방문**
   - https://obsidian.md/ 접속
   - Windows용 설치 파일 다운로드

2. **설치 파일 실행**
   - 다운로드한 .exe 파일 실행
   - 설치 마법사 지침 따라 설치 완료

### 🔹 2.2 볼트(Vault) 생성

처음 실행 시 **'Create new vault'** 버튼을 클릭하여 새로운 볼트를 생성할 수 있습니다.

**볼트(Vault)**는 옵시디언에서 노트를 저장하는 기본 단위입니다. 각 볼트는 독립적인 폴더이며, 하나의 프로젝트나 주제별로 볼트를 분리하여 관리할 수 있습니다.

```
볼트 구조 예시:
MyVault/
├── .obsidian/          # 설정 파일
│   ├── plugins/        # 플러그인
│   ├── themes/         # 테마
│   └── snippets/       # CSS 스니펫
├── Daily Notes/        # 일간 노트
├── Projects/           # 프로젝트 노트
└── Resources/          # 참고 자료
```

### 🛠️ 2.3 초기 언어 설정

1. **Settings(설정)** 열기 (단축키: `Ctrl + ,`)
2. **Appearance** 탭 선택
3. **Language** 드롭다운에서 **한국어** 선택
4. Obsidian 재시작

## 🛠️ 3. 추천 설정

### 🛠️ 3.1 외관(Appearance) 설정

#### ▫️ 테마 변경
1. **설정 > 외관** 이동
2. **테마** 섹션에서 **관리** 클릭
3. 커뮤니티 테마 목록에서 원하는 테마 선택 및 설치

**추천 테마:**
- **Minimal** - 깔끔하고 미니멀한 디자인
- **California Coast** - 부드러운 색상 조합
- **Things** - Things 앱 스타일의 테마

#### ▫️ 다크/라이트 모드 전환
- **설정 > 외관 > 기본 색상 모드**
- 자동/라이트/다크 중 선택

### 🛠️ 3.2 편집기(Editor) 설정

#### ▫️ 읽기 쉬운 줄 길이
- **설정 > 편집기 > 읽기 쉬운 줄 길이** 활성화
- 긴 텍스트 줄을 읽기 편한 폭으로 제한

#### ▫️ 자동 쌍 괄호
- **설정 > 편집기 > 자동 쌍 괄호** 활성화
- `(`, `[`, `{`, `"` 입력 시 자동으로 닫는 괄호 추가

### 🗂️ 3.3 파일 및 링크 설정

#### ▫️ 새 링크 형식
- **설정 > 파일 및 링크 > 새 링크 형식**
- **절대 경로** 또는 **상대 경로** 선택

#### 🗂️ 첨부 파일 기본 위치
- **설정 > 파일 및 링크 > 첨부 파일 기본 위치**
- 이미지 및 파일 첨부 시 저장 위치 지정

### 🛠️ 3.4 동기화 설정

#### ▫️ iCloud를 통한 무료 동기화
1. 볼트를 **iCloud Drive의 Obsidian 폴더**에 저장
2. 다른 기기(iOS, macOS)에서 동일한 iCloud 계정으로 접속
3. 자동 동기화 완료

#### ▫️ Git을 통한 동기화
- **Obsidian Git** 플러그인 설치
- GitHub/GitLab 저장소와 연동하여 버전 관리 및 동기화

## 🛠️ 4. 특정 원하는 설정

### 🔹 4.1 MD Preview 뷰어 Full Width로 보기 (좌우 마진 제거)

기본적으로 Obsidian의 프리뷰 모드는 읽기 쉬운 줄 길이를 유지하기 위해 좌우 마진을 두고 있습니다. 이를 제거하고 전체 너비로 보려면 **CSS 스니펫**을 사용해야 합니다.

#### 🛠️ 설정 방법:

**1단계: CSS 스니펫 폴더 열기**
- **설정 > 외관 > CSS 스니펫** 이동
- 폴더 아이콘 클릭하여 **snippets 폴더** 열기

**2단계: CSS 파일 생성**
- `full-width.css` 파일 생성

**3단계: CSS 코드 입력**
```css
/* Full Width Preview - 좌우 마진 제거 */
.markdown-preview-view {
  max-width: 100% !important;
  width: 100% !important;
  padding-left: 10px !important;
  padding-right: 10px !important;
}

/* Reading View Full Width */
.markdown-reading-view {
  max-width: 100% !important;
  width: 100% !important;
}

/* Source Mode Full Width */
.markdown-source-view {
  max-width: 100% !important;
  width: 100% !important;
}

/* 읽기 쉬운 줄 길이 설정 무시 */
.markdown-preview-section {
  max-width: 100% !important;
}
```

**4단계: CSS 스니펫 활성화**
- Obsidian으로 돌아가서 **설정 > 외관 > CSS 스니펫**
- 방금 생성한 `full-width.css` 토글 활성화

### 🔹 4.2 철자 오류 탐색 비활성화

Obsidian은 영어와 한국어 맞춤법 검사를 지원하지만, 코드나 전문 용어 작성 시 방해가 될 수 있습니다.

#### ▫️ 비활성화 방법:

**1단계: 설정 열기**
- 단축키 `Ctrl + ,` 또는 좌측 하단 톱니바퀴 아이콘 클릭

**2단계: 코어 플러그인 비활성화**
- **설정 > 코어 플러그인(Core plugins)** 이동
- **맞춤법 검사(Spell checker)** 찾기
- 토글을 **꺼짐(OFF)** 상태로 변경

```
설정 경로:
Settings → Core plugins → Spell checker [OFF]
```

### 🔹 4.3 Font: D2Coding으로 변경

**D2Coding**은 개발자를 위한 한글/영문 코딩용 폰트로, 가독성이 뛰어나며 한글과 영문의 조화가 좋습니다.

#### 🛠️ 설정 방법:

**방법 1: Custom Font Loader 플러그인 사용 (추천)**

**1단계: D2Coding 폰트 다운로드**
- 네이버 D2Coding 폰트 다운로드 페이지 방문
- `.ttf`, `.woff`, 또는 `.woff2` 형식 파일 다운로드

**2단계: 폰트 파일 배치**
- 볼트 내 `.obsidian/fonts` 폴더 생성 (없는 경우)
- 다운로드한 D2Coding 폰트 파일을 이 폴더에 복사

```
경로 예시:
MyVault/.obsidian/fonts/D2Coding.ttf
```

**3단계: Custom Font Loader 플러그인 설치**
- **설정 > 커뮤니티 플러그인 > 찾아보기**
- **Custom Font Loader** 검색 및 설치
- 플러그인 활성화

**4단계: 폰트 설정**
- Custom Font Loader 플러그인 설정 열기
- 드롭다운 메뉴에서 **D2Coding** 선택
- 텍스트 폰트와 모노스페이스 폰트 모두 D2Coding으로 설정

**방법 2: CSS 스니펫 사용**

**1단계: CSS 스니펫 생성**
- `.obsidian/snippets/d2coding-font.css` 파일 생성

**2단계: CSS 코드 입력**
```css
/* D2Coding Font 적용 */
@font-face {
  font-family: "D2Coding";
  src: url("app://local/.obsidian/fonts/D2Coding.ttf") format("truetype");
}

/* 전체 에디터에 D2Coding 폰트 적용 */
body {
  --font-text: "D2Coding", -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif !important;
  --font-monospace: "D2Coding", "Consolas", "Monaco", monospace !important;
}

/* 편집 모드 */
.cm-content {
  font-family: "D2Coding" !important;
}

/* 미리보기 모드 */
.markdown-preview-view {
  font-family: "D2Coding" !important;
}

/* 코드 블록 */
code, pre {
  font-family: "D2Coding" !important;
}
```

**3단계: CSS 스니펫 활성화**
- **설정 > 외관 > CSS 스니펫**
- `d2coding-font.css` 토글 활성화

**방법 3: 시스템 폰트로 설정**

**1단계: D2Coding 폰트 시스템에 설치**
- Windows 설정 > 개인 설정 > 글꼴
- 다운로드한 D2Coding 폰트 파일 드래그 & 드롭으로 설치

**2단계: Obsidian 설정**
- **설정 > 외관**
- **텍스트 글꼴(Text font)**: `D2Coding` 입력
- **모노스페이스 글꼴(Monospace font)**: `D2Coding` 입력

```
설정 경로:
Settings → Appearance → Text font: "D2Coding"
Settings → Appearance → Monospace font: "D2Coding"
```

**참고:** D2Coding 폰트는 영어로만 인식되므로, 폰트 이름을 정확히 **"D2Coding"**으로 입력해야 합니다.

## 📌 5. 자주 사용하는 단축키 (Windows)

| 단축키 | 기능 | 설명 |
|--------|------|------|

### 🔹 5.5 고급 기능

### 🛠️ 5.6 커스텀 단축키 설정

Obsidian에서는 모든 단축키를 사용자 정의할 수 있습니다.

**설정 방법:**
1. **설정 > 단축키(Hotkeys)** 이동
2. 검색창에 원하는 기능 입력
3. 빈 단축키 항목 클릭 후 원하는 키 조합 입력
4. 충돌하는 단축키가 있으면 경고 표시

**추천 커스텀 단축키:**
- **Daily Note 열기**: `Ctrl + Shift + D`
- **템플릿 삽입**: `Ctrl + T`
- **파일 탐색기 포커스**: `Ctrl + Shift + E`

## 📌 6. 필수 플러그인 추천

### 🛠️ 6.1 플러그인 설치 방법

**1단계: 커뮤니티 플러그인 활성화**
- **설정 > 커뮤니티 플러그인**
- **제한 모드 끄기(Turn off Restricted mode)** 클릭

**2단계: 플러그인 찾아보기**
- **찾아보기(Browse)** 클릭
- 플러그인 이름 검색

**3단계: 설치 및 활성화**
- 원하는 플러그인 선택
- **설치(Install)** → **활성화(Enable)** 클릭

### 🔹 6.2 편집 관련 플러그인

#### ▫️ Editing Toolbar
- **기능:** 마크다운 문법을 몰라도 버튼 클릭으로 서식 적용
- **활용:** 글자 색상, 배경색, 정렬 등을 직관적으로 설정
- **추천 대상:** 마크다운 초보자, 빠른 서식 적용이 필요한 사용자

## 🏢 7. 그래프 뷰 활용법

### 🔹 7.1 그래프 뷰란?

**그래프 뷰(Graph View)**는 Obsidian의 핵심 기능 중 하나로, 노트 간의 연결을 시각적으로 보여주는 도구입니다. 생성된 노트가 점(노드)의 형태로 보이며, 노트 간의 링크가 선으로 연결되어 표시됩니다.

### 🔹 7.2 그래프 뷰 열기

**방법 1: 사이드바 버튼**
- 좌측 사이드바에서 **"Open Graph View"** 버튼 클릭

**방법 2: 명령어 팔레트**
- `Ctrl + P`로 명령어 팔레트 열기
- "Open Graph View" 검색 및 선택

**방법 3: 단축키 설정**
- **설정 > 단축키**에서 "Graph View" 검색
- 원하는 단축키 할당 (예: `Ctrl + G`)

### 🔹 7.3 노트 연결하기

그래프 뷰에서 연결을 만들려면 **`[[ ]]`** 문법을 사용합니다.

```markdown
## 예시:
Obsidian은 [[마크다운]] 기반의 노트 앱입니다.
[[그래프 뷰]]를 통해 노트 간의 연결을 시각화할 수 있습니다.

## 결과:
현재 노트와 "마크다운", "그래프 뷰" 노트가 그래프에서 연결됨
```

### 🔹 7.4 그래프 뷰 조작

| 조작 | 기능 |
|------|------|
| **클릭** | 노트 열기 |
| **드래그** | 그래프 이동 |
| **마우스 휠** | 줌 인/줌 아웃 |
| **노드 드래그** | 특정 노드 위치 이동 |
| **더블 클릭** | 노드에 포커스 |

### 🔹 7.5 그래프 뷰 필터링

그래프 뷰 우측에는 다양한 필터링 옵션이 있습니다:

**필터(Filters):**
- 검색 쿼리를 사용하여 특정 노트만 표시
- 예: `tag:#important` - #important 태그가 있는 노트만 표시

**그룹(Groups):**
- 특정 조건의 노트를 색상으로 구분
- 예: 프로젝트별, 태그별, 폴더별 색상 구분

**표시(Display):**
- 화살표 표시 여부
- 텍스트 페이드 임계값
- 노드 크기 조절

**힘(Forces):**
- 중심력, 반발력, 링크 길이, 링크 거리 조절
- 그래프의 물리적 레이아웃 커스터마이징

### 🔹 7.6 로컬 그래프 뷰

**로컬 그래프 뷰(Local Graph View)**는 현재 열린 노트와 직접 연결된 노트만 보여주는 기능입니다.

**열기 방법:**
- 우측 사이드바의 **"Open local graph"** 아이콘 클릭
- 명령어 팔레트에서 "Open local graph" 선택

**활용:**
- 특정 노트와 관련된 내용만 집중하여 탐색
- 복잡한 전체 그래프 대신 관련 노트만 확인

### 🏢 7.7 그래프 뷰 실전 활용 팁

**1. 태그를 활용한 주제별 분류**
```markdown
#프로젝트/메피아
#학습/Obsidian
#아이디어
```

**2. 허브 노트 만들기**
- 특정 주제의 중심이 되는 노트를 만들고, 관련 노트들을 모두 링크
- 그래프 뷰에서 큰 노드로 표시되어 중요도를 시각적으로 파악

**3. 백링크(Backlinks) 활용**
- 다른 노트에서 현재 노트로 연결된 링크 확인
- 노트 우측 사이드바의 "Backlinks" 섹션 참고

**4. 고아 노트 찾기**
- 그래프 뷰에서 연결되지 않은 독립된 노트(고아 노트) 발견
- 이러한 노트들을 적절히 연결하여 지식 네트워크 강화
