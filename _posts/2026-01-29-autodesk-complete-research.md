## 🧭 상세 소개

### 🕰️ 회사의 역사와 발전

Autodesk는 1982년 존 워커(John Walker)를 포함한 12명의 공동 창립자에 의해 설립되었습니다. 회사의 대표 CAD 소프트웨어 제품인 AutoCAD의 초기 버전 공동 제작자들이 캘리포니아 산 라파엘에 본사를 설립하면서 시작되었습니다.

#### ▫️ 초기 성공 (1982-1989)
- **1982년 12월**: AutoCAD 첫 출시, 첫 해에 140만 달러의 수익 달성
- **1984년**: 회사 설립 첫 달부터 수익을 창출하며 대출이나 외부 투자자 없이 성장
- **1985년**: 나스닥에 상장
- **1989년**: 단 4년의 운영 기간 만에 매출이 1억 달러 돌파

#### ▫️ 확장과 다각화 (1990년대-2000년대)
Autodesk는 초기 AutoCAD의 성공을 기반으로 다양한 산업 분야로 제품군을 확장했습니다:
- 건축 및 건설 분야로 진출 (Revit 인수)
- 제조 및 제품 설계 분야 강화 (Inventor 개발)
- 미디어 & 엔터테인먼트 분야 진입 (Maya, 3ds Max 인수)

#### ▫️ 현대의 Autodesk (2010년대-현재)
- 클라우드 기반 협업 플랫폼 개발
- 구독 모델로의 전환 (2016년)
- AI 및 제너레이티브 디자인 기술 통합
- Autodesk Platform Services (구 Forge) 출시

### 🔹 기업 비전과 미션

Autodesk의 핵심 비전은 **"Make anything possible"**입니다. 모든 혁신가들이 자신의 아이디어를 현실화할 수 있도록 돕는 것을 목표로 하며, 디자인과 제작(Design and Make) 과정을 디지털화하고 자동화하여 더 나은 세상을 만드는 데 기여하고자 합니다.

## 🎯 Autodesk의 필요성

### 🔹 1. 디지털 전환의 핵심 도구

현대 산업에서 디지털 트랜스포메이션은 선택이 아닌 필수입니다. Autodesk의 소프트웨어는 다음과 같은 이유로 필수적입니다:

- **정확성과 정밀성**: 수작업 도면에서 발생하는 오류를 최소화
- **효율성 향상**: 반복 작업 자동화로 설계 시간 단축
- **협업 강화**: 클라우드 기반 플랫폼으로 실시간 협업 가능
- **비용 절감**: 물리적 프로토타입 제작 전 디지털 시뮬레이션으로 비용 절감

### 🔹 3. 통합된 워크플로우

Autodesk 제품군은 설계부터 제조, 유지보수까지 전체 라이프사이클을 커버합니다:

1. **개념화 단계**: 스케치, 아이디어 구상
2. **설계 단계**: 2D/3D 모델링, 시뮬레이션
3. **협업 단계**: 팀 간 실시간 협업, 검토
4. **제작 단계**: CAM, CNC 프로그래밍
5. **분석 단계**: 구조 분석, 성능 시뮬레이션
6. **문서화 단계**: 도면, 시방서 자동 생성
7. **운영 단계**: 시설 관리, 유지보수 계획

## 📌 사용 분야

Autodesk의 소프트웨어는 거의 모든 산업 분야에서 활용되고 있습니다. 아래는 주요 사용 분야별 상세 설명입니다.

### ⚠️ 1. 건축, 엔지니어링, 건설 (AEC - Architecture, Engineering, Construction)

#### ▫️ 건축 설계
- **주요 도구**: AutoCAD, Revit, Forma
- **활용**: 건물 설계, 평면도, 입면도, 단면도 작성
- **실제 사례**: 베이징 신축 서브 센터 극장 단지는 건축부터 음향 시설까지 모든 것을 완벽하게 구현하기 위해 스마트 건설 기술을 사용했습니다

#### 🏢 BIM (Building Information Modeling)
- **주요 도구**: Revit, BIM 360, Autodesk Construction Cloud
- **장점**:
  - 프로젝트 타임라인 평균 20% 단축
  - 비용 15% 절감
  - 설계 오류 30% 감소

### 🔹 2. 제조 및 제품 설계

#### ▫️ 기계 설계
- **주요 도구**: Inventor, Fusion 360
- **활용**: 제품 프로토타입, 기계 부품 설계, 조립 시뮬레이션
- **파일 형식**: IPT (Inventor Part), IAM (Inventor Assembly)
- **실제 사례**: 냉난방 전문업체 비스만(Viessmann)은 새로운 히트펌프 스마트 팩토리의 생산량을 늘리기 위해 디지털 팩토리 계획을 사용

#### ▫️ 제너레이티브 디자인
- **주요 도구**: Fusion 360 (Generative Design Extension)
- **AI 활용**: 수천 개의 디자인 대안을 자동으로 생성
- **장점**:
  - 소재 효율성 극대화 (강도 유지하면서 재료 사용량 감소)
  - 설계 프로세스 속도 향상
  - 혁신적인 솔루션 발견

#### 💻 CAM (Computer-Aided Manufacturing)
- **주요 도구**: Fusion 360, FeatureCAM
- **활용**: CNC 공작기계 프로그래밍, 툴패스 생성

### 🔹 3. 미디어 & 엔터테인먼트

#### ▫️ 영화 및 VFX
- **주요 도구**: Maya, 3ds Max, Arnold (렌더러)
- **Academy Award**: Maya는 아카데미상을 수상한 3D 디자인 소프트웨어
- **실제 사례**:
  - 영화 **Avatar**에서 MotionBuilder를 사용하여 배우의 퍼포먼스를 캡처하고 3D 캐릭터로 전송
  - 헐리우드 대부분의 블록버스터 영화 CG와 VFX 제작에 Maya 활용

#### ▫️ 게임 개발
- **주요 도구**: Maya, 3ds Max, MotionBuilder
- **활용**: 캐릭터 모델링, 애니메이션, 리깅, 환경 디자인

#### ▫️ 건축 시각화
- **주요 도구**: 3ds Max, V-Ray
- **활용**: 포토리얼리스틱 렌더링, 건축 프레젠테이션

### 🔹 4. 산업 간 협업

Autodesk의 독특한 강점 중 하나는 산업 간 데이터 공유와 협업을 가능하게 한다는 점입니다. 영화와 건축, 엔지니어링, 건설 산업은 유사한 도구, 프로세스, 워크플로를 사용하며, 이러한 유사성은 다양한 프로젝트에 가치를 더할 수 있는 3D 자산 등의 데이터를 공유할 기회를 제공합니다.

## 📌 주요 제품군

Autodesk는 2024년 9월 기준으로 **112개의 소프트웨어 제품**을 보유하고 있으며, 이를 크게 3개의 산업별 컬렉션으로 분류합니다.

### ⚠️ 1. Architecture, Engineering & Construction (AEC) Collection

#### ▫️ AutoCAD
- **출시**: 1982년 (최신 버전: AutoCAD 2026)
- **용도**: 2D/3D CAD 설계, 제도, 문서화
- **주요 특징**:
  - 수백만 명의 신뢰를 받는 정밀 제도 소프트웨어
  - 산업 표준 DWG 파일 형식
- **AutoCAD 2026 신규 기능**:
  - AI 기반 "스마트 블록: 검색 및 변환" 기능
  - Autodesk Assistant: 대화형 AI 어시스턴트
  - 성능 향상: 파일 열기 속도 최대 11배, 프로그램 구동 속도 최대 4배 개선
  - 멀티 사용자 마크업 기능 (웹 버전)

#### ▫️ Revit
- **용도**: BIM (Building Information Modeling)
- **활용 분야**:
  - 건축 설계
  - 구조 엔지니어링
  - MEP (Mechanical, Electrical, Plumbing) 설계
  - 시공 관리
- **파일 형식**: RVT (Revit Project), RTE (Revit Template), RFA (Revit Family)
- **주요 특징**:
  - 파라메트릭 모델링
  - 자동 문서화 (평면도, 입면도, 단면도, 일람표 자동 생성)
  - 협업 기능 (Worksharing)

#### ⚠️ Autodesk Construction Cloud (ACC)
- **구성**: BIM Collaborate, Docs, Build, Takeoff
- **이전 제품**: BIM 360 (현재 ACC로 통합 중)
- **차이점**:
  - **BIM 360**: 대규모 건설 프로젝트에 초점
  - **ACC**: 소규모 및 대규모 프로젝트 모두 지원, 비용 관리, 현장 관리, 안전 관리 포함
- **현재 상황**: Autodesk는 더 이상 BIM 360 개발에 집중하지 않고 ACC로 전환 중

#### ▫️ Civil 3D
- **용도**: 토목 설계 및 문서화
- **활용**: 도로, 교량, 터널, 토지 개발

#### ▫️ Forma
- **용도**: 초기 단계 사이트 계획 및 디자인
- **주요 특징**: 제너레이티브 디자인과 AI를 활용한 최적화

### 🔹 2. Product Design & Manufacturing Collection

#### ▫️ Autodesk Inventor
- **출시**: 1999년
- **용도**: 파라메트릭 3D CAD 기계 설계
- **파일 형식**:
  - IPT (Inventor Part): 개별 부품
  - IAM (Inventor Assembly): 조립품
- **지오메트릭 커널**: ShapeManager (ACIS 기반)
- **주요 기능**:
  - 피처 히스토리
  - 파라메트릭 제약조건
  - 3D 주석 (PMI - Product Manufacturing Information)
  - 파이프, 튜브, 와이어 등 기계 부품 모델링

#### ▫️ Fusion 360
- **플랫폼**: 클라우드 기반
- **통합 기능**: CAD + CAM + CAE + PCB 설계
- **주요 특징**:
  - 제너레이티브 디자인
  - 시뮬레이션
  - CNC 가공 프로그래밍
  - 협업 기능
- **타겟**: 제품 디자이너, 기계 엔지니어, 스타트업
- **가격 모델**: 개인용 무료 버전 제공 (연 수익 10만 달러 미만)

#### ✨ FeatureCAM, PowerMill
- **용도**: 고급 CAM 소프트웨어
- **활용**: 복잡한 CNC 가공, 5축 가공

#### ▫️ Vault
- **용도**: 제품 데이터 관리 (PDM)
- **활용**: 버전 관리, 파일 체크인/체크아웃

## 🗂️ 파일 형식

Autodesk는 다양한 독점 파일 형식과 오픈 표준 파일 형식을 지원합니다. 아래는 주요 파일 형식에 대한 상세 설명입니다.

### 🗂️ 1. CAD 파일 형식

#### ▫️ DWG (Drawing)
- **소유**: Autodesk 독점 형식
- **용도**: AutoCAD 기본 파일 형식
- **특징**:
  - 이진 코드 (Binary) 사용
  - DXF보다 약 25% 작은 파일 크기
  - 고해상도 그래픽과 복잡한 도면 정보 효율적 저장
- **버전**: AutoCAD 버전마다 DWG 형식 버전이 다름 (하위 호환성 고려 필요)
- **구조**:
  - Header (파일 메타데이터)
  - Classes (객체 클래스 정의)
  - Objects (그래픽 및 비그래픽 객체)

```
DWG File Structure:
┌─────────────────────────┐
│  File Header            │ ← File version, metadata
├─────────────────────────┤
│  Classes Section        │ ← Custom object classes
├─────────────────────────┤
│  Objects Section        │
│  ┌───────────────────┐  │
│  │ Entity Data       │  │ ← Lines, circles, etc.
│  ├───────────────────┤  │
│  │ Symbol Tables     │  │ ← Layers, linetypes
│  ├───────────────────┤  │
│  │ Block Definitions │  │ ← Reusable blocks
│  └───────────────────┘  │
├─────────────────────────┤
│  Image Data             │ ← Thumbnail preview
└─────────────────────────┘
```

#### ▫️ DXF (Drawing Exchange Format)
- **출시**: 1982년
- **용도**: CAD 프로그램 간 도면 데이터 교환
- **특징**:
  - 텍스트 기반 (ASCII) 또는 이진 형식 지원
  - 오픈 소스 (공개 스펙)
  - 다른 CAD 소프트웨어와 호환성 우수
- **구조**:
  - **HEADER 섹션**: 도면에 대한 일반 정보
  - **CLASSES 섹션**: 응용프로그램 정의 클래스 정보
  - **TABLES 섹션**: 기호 테이블 정의 (레이어, 선종류, 블록 등)
  - **BLOCKS 섹션**: 블록 정의 및 도면 요소
  - **ENTITIES 섹션**: 도면의 그래픽 객체

### 🗂️ 2. BIM 파일 형식

#### ▫️ RVT (Revit Project)
- **소유**: Autodesk 독점 형식
- **용도**: Revit 프로젝트 파일
- **내용**:
  - 3D 건물 모델
  - 평면도, 입면도, 단면도
  - 일람표 (Schedule)
  - 메타데이터 (재료, 비용, 제작사 정보 등)
- **특징**:
  - 파라메트릭 관계 저장
  - 모든 뷰가 단일 모델에서 자동 생성
  - BIM 데이터 (기하학, 관계, 속성) 포함

#### ▫️ RFA (Revit Family)
- **용도**: 재사용 가능한 Revit 컴포넌트
- **예시**: 문, 창문, 가구, MEP 장비
- **특징**: 파라메트릭 정의로 크기와 속성 조정 가능

#### ▫️ RTE (Revit Template)
- **용도**: Revit 프로젝트 템플릿
- **내용**: 기본 설정, 뷰 템플릿, 패밀리

#### ▫️ IFC (Industry Foundation Classes)
- **표준**: ISO 16739
- **용도**: BIM 소프트웨어 간 데이터 교환
- **지원**: Revit에서 IFC 가져오기/내보내기 지원

### 🗂️ 3. 제조 및 제품 설계 파일 형식

#### ▫️ IPT (Inventor Part)
- **용도**: Inventor 부품 파일
- **내용**:
  - 2D 또는 3D 파트
  - 피처 히스토리
  - 파라메트릭 제약조건
  - PMI (Product Manufacturing Information)
- **지오메트릭 커널**: ShapeManager (ACIS 기반)

#### ▫️ IAM (Inventor Assembly)
- **용도**: Inventor 조립품 파일
- **내용**:
  - 여러 IPT 파일의 조립
  - 제약조건 (Constraints)
  - 어셈블리 피처

#### ▫️ F3D (Fusion 360 Archive)
- **용도**: Fusion 360 프로젝트 파일
- **특징**: 클라우드 기반이지만 로컬 아카이브 가능

### 🗂️ 4. 3D 및 애니메이션 파일 형식

#### ▫️ FBX (Filmbox)
- **소유**: Autodesk (오픈 프레임워크)
- **용도**: 3D 데이터 전송 및 상호 운용성
- **지원 데이터**:
  - 기하학 (Geometry)
  - 애니메이션 (키프레임, 모션 캡처)
  - 카메라
  - 조명
  - 리깅 (Skeleton)
  - 텍스처
- **형식**: 바이너리 또는 ASCII
- **압축**: 내부 데이터 압축 지원
- **활용 분야**:
  - VFX 파이프라인
  - 게임 엔진 (Unity, Unreal Engine)
  - Maya, 3ds Max, MotionBuilder 간 데이터 교환

```

│  Definitions (Nodes)                │
│  ┌───────────────────────────────┐  │
│  │ Model (Mesh)                  │  │
│  │ ├─ Geometry                   │  │
│  │ ├─ Material                   │  │
│  │ └─ Texture                    │  │
│  │                               │  │
│  │ AnimationStack                │  │
│  │ ├─ AnimationLayer             │  │
│  │ └─ AnimationCurve             │  │
│  │                               │  │
│  │ Skeleton                      │  │
│  │ ├─ Joint hierarchy            │  │
│  │ └─ Skin weights               │  │
│  └───────────────────────────────┘  │
├─────────────────────────────────────┤
│  Connections (Relationships)        │
└─────────────────────────────────────┘
```

#### ▫️ MA (Maya ASCII) / MB (Maya Binary)
- **용도**: Maya 씬 파일
- **MA**: 텍스트 기반 (버전 관리 용이)
- **MB**: 이진 형식 (파일 크기 작음)

#### ▫️ MAX
- **용도**: 3ds Max 씬 파일
- **특징**: Windows 전용

## 📌 라이선스 정책

Autodesk는 2016년부터 영구 라이선스를 중단하고 구독(Subscription) 모델로 완전히 전환했습니다.

### 🧭 1. 구독 모델 개요

#### ▫️ 전환 타임라인
```
Autodesk Licensing History:
1982-2015: Perpetual License (영구 라이선스)
            ↓
2016:       Subscription model introduced
            (구독제 도입, 영구 라이선스 병행)
            ↓
2017:       AutoCAD subscription-only
            (오토캐드 구독제 전용)
            ↓
2020:       All perpetual licenses discontinued
            (모든 영구 라이선스 중단)
            ↓
2023 Feb 7: Network licenses discontinued
            (네트워크 라이선스 완전 종료)
            ↓
2023-현재:  Single-user subscription only
            (싱글 사용자 구독제만 제공)
```

- **월간 구독** (Monthly)
